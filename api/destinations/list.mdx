title: 'List Destinations'

## Authentication

**Required Role:** None (Public endpoint)

This endpoint is publicly accessible and does not require authentication.

## Response

<ResponseField name="total" type="number">
  Total count of active destinations
</ResponseField>

<ResponseField name="data" type="array">
  Array of destination objects
  
  <ResponseField name="data[]._id" type="string">
    MongoDB ObjectId of the destination
  </ResponseField>
  
  <ResponseField name="data[].nombre" type="string">
    Name of the heritage site
  </ResponseField>
  
  <ResponseField name="data[].ubicacion" type="string">
    Location or address
  </ResponseField>
  
  <ResponseField name="data[].importancia" type="string">
    Historical or cultural significance
  </ResponseField>
  
  <ResponseField name="data[].coordenadas" type="object">
    Geographic coordinates
    
    <ResponseField name="data[].coordenadas.latitud" type="number">
      Latitude
    </ResponseField>
    
    <ResponseField name="data[].coordenadas.longitud" type="number">
      Longitude
    </ResponseField>
  </ResponseField>
  
  <ResponseField name="data[].anio_construccion" type="array">
    Year(s) of construction (1 or 2 values)
  </ResponseField>
  
  <ResponseField name="data[].arquitecto" type="string">
    Architect name (if available)
  </ResponseField>
  
  <ResponseField name="data[].area_construccion" type="number">
    Construction area in square meters (if available)
  </ResponseField>
  
  <ResponseField name="data[].funcion" type="string">
    Function of the site (if available)
  </ResponseField>
  
  <ResponseField name="data[].multimedia" type="array">
    Array of multimedia files associated with this destination
    
    <ResponseField name="data[].multimedia[].url" type="string">
      Cloudinary secure URL for the file
    </ResponseField>
    
    <ResponseField name="data[].multimedia[].public_id" type="string">
      Cloudinary public ID
    </ResponseField>
    
    <ResponseField name="data[].multimedia[].tipo" type="string">
      File type: "image" or "video"
    </ResponseField>
    
    <ResponseField name="data[].multimedia[].formato" type="string">
      File format (e.g., "jpg", "png", "mp4")
    </ResponseField>
    
    <ResponseField name="data[].multimedia[].tamanio_bytes" type="number">
      File size in bytes
    </ResponseField>
    
    <ResponseField name="data[].multimedia[].anchura" type="number">
      Width in pixels (for images/videos)
    </ResponseField>
    
    <ResponseField name="data[].multimedia[].altura" type="number">
      Height in pixels (for images/videos)
    </ResponseField>
    
    <ResponseField name="data[].multimedia[].duracion" type="number">
      Duration in seconds (for videos only)
    </ResponseField>
  </ResponseField>
  
  <ResponseField name="data[].fecha_creacion" type="string">
    ISO 8601 timestamp of creation
  </ResponseField>
  
  <ResponseField name="data[].activo" type="boolean">
    Always true (only active destinations are returned)
  </ResponseField>
</ResponseField>

<RequestExample>

```bash cURL
curl https://api.tesisrutas.com/destinos
```

```python Python
import requests

url = "https://api.tesisrutas.com/destinos"
response = requests.get(url)

data = response.json()
print(f"Total destinations: {data['total']}")
for destino in data['data']:
    print(f"- {destino['nombre']} ({destino['ubicacion']})")
```

```javascript JavaScript
const response = await fetch('https://api.tesisrutas.com/destinos');
const data = await response.json();

console.log(`Total destinations: ${data.total}`);
data.data.forEach(destino => {
  console.log(`- ${destino.nombre} (${destino.ubicacion})`);
});
```

## Implementation Details

This endpoint:
1. Queries the `sitios_patrimoniales` collection with filter `{"activo": true}`
2. Returns only destinations marked as active
3. Converts MongoDB ObjectIds to strings for JSON serialization
4. Includes all associated multimedia files in the response

**Filter Behavior:**
- Only destinations with `activo: true` are returned
- Deactivated destinations (toggled via `/estado/{id}`) are excluded
- No pagination is currently implemented
